# Deep Dive Tech Docs
> 개발 관련 영어 문서 읽기 스터디

## 진행 방식
- 진행 기간: 06/14(일) ~ 07/12(일)
- 정기 모임: 매주 일요일 10시 ~ 13시
- 모임 방식: 온라인
- 진행 방식:
    - 10:00 ~ 12:00 | 번역 및 공부 + 번역한 내용 Github에 업로드
    - 12:00 ~ 13:00 | 개인 발표

## 번역 및 공부
- 브랜치 생성한 후 본인 폴더 하위에 파일 만들어서 번역
- 브랜치 규칙: {이름 이니셜}-주제
  - 예시: kyj-spring-transaction
- 커밋: {이름 이니셜}: 문서내용
  - 예시: kyj:spring transaction
- 파일명은 자유
- PR은 템플릿 맞춰서 12시까지 올리기
- 띄어쓰기 필요할 경우 모두 하이픈(-)으로 통일

## 발표 방식
- 백지 복습. 미리 자료 준비 X. 
- 메모장 또는 그림판 등 빈 화면에 본인이 번역한 글에 대한 공부 내용을 적으면서 설명해주기
